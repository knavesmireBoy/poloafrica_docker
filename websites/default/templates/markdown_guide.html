• The Markdown Guide by Matt Cone
  • The toolbar above provides some handy shortcuts for creating markdown markup. For a comprehensive guide click the link above.
  • For inline formatting: For a single word selection just place the cursor within the word and click to apply the required formatting, otherwise start and end your selection anywhere within the first and last words.
  • The H button will with each click add a hash to the start of the line, returning to one hash at the seventh succesive click, The result would be a heading from h1 thru h6.
  • The B button will toogle a double pair of asterisks on the selected text. For a single word simply place the cursor anywhwere in the word and click. The result would be to embolden the text. To use an actual asterisk use the HTML entity "&ast;" eg **my footnote&ast;**, for bold asterisk, or **my footnote**&ast; for normal.
  • The I button will toogle a single pair of asterisks on the selected text. For a single word see above. The result would be to italicise the text. Already emboldened text will be respected. Therefore my bold text would become my bold text.
  • The ¶ inserts a blank line. Place cursor in a sentence ending in a full stop (period) and click. A new paragraph is created. Alternatively simply type a return at the end of the sentence.
  • The • adds an html break tag. Place cursor in a sentence ending in a full stop (period) and click.
  • The </> sets/unsets any legitimate inline html tag (excluding those tags already catered for and also self-closing tags). It invokes a prompt button to allow you to type in the tag name, defaulting to span. It otherwise behaves as the bold/italic button. Make a selection to surround multiple words, or place cursor in target word for a single word. To remove the inline tag straddle the range of words and click. The same thing can obviously be achieved manually and in most cases you may wish to furnish the tag with a relevant attribute. Directly using HTML in Markdown applications however does not have 100% support, again see the guide for details. Useful lessons can also be found here.
  • The link button creates a link on the selected text. A confirm dialog window opens. Enter a genuine link to either an external website ("https://www.bbc.co.uk/") or an internal web page ("/enquiries/"), or to a resource (/resource/assets/medley/polotimes.pdf). To add a title to the link add one space after the hyperlink followed by the title content. By default all links will open in a separate browser window. This feature will break if no heading was present at the top of the article, so an attempt to add a link will first create a heading from the title field to fix this issue.
  • The unlink button clears the link. Click anywhere in the FIRST word [Poloafrica Development Trust][2]" to clear.
  • The list button converts a selection of lines into an unordered list:
    1. • a
    2. • b
    3. • c
    Subsequent clicks will cycle between no list (paragraphs) and a list. With a list alternating between an ordered list and an unordered list.
  • The image button adds an image. This would normally never be required as images should be independent of the article text and can be managed by clicking the icons below the submit button. However it may be useful for the occasional inline icon. All images must have an alt attribute even if it is an empty string. Type in any alterate text, click anywhere in the line, a dialog box will open. Enter the path to the image and an optional title. See link above. Images will be placed INLINE at the chosen location, unless it is desirable to have the image in its own paragraph, in which case follow the image with a hard return.
  • The arrow button CLEARS all edits.
  • The ? button toggles this guide. As will clicking on this line.
